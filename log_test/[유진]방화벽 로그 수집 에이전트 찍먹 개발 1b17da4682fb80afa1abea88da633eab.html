로그 수집 에이전트 코드

파일명 : log_test.py

import time 

LOG_FILE="/var/log/forigate.log"

# 수집한 로그를 원하는 대로 처리하는 함수
def process_log(log_data):
    if "attack" in log_data or "blocked" in log_data:
        print("[Warring! 경고 로그 감지]", log_data.strip())


# 실시간으로 로그 파일을 감시하여 새로운 로그가 추가되면 처리
def monitor_log():
    with open(LOG_FILE, "r") as flie:
        flie.seek(0, 2)     # 파일 끝으로 이동
        while True:
            line = flie.readline()
            if not line:
                time.sleep(0.1)
                continue
            process_log(line)

if __name__=="__main__":
    monitor_log()

1. 로그 수집 에이전트를 자동 실행되도록 설정하기 위해 systemd 서비스를 설정

a. 서비스 파일 생성 및 작성

vi /etc/systemd/system/fortigate_log_collector.service

[Unit]
Description=FortiGate Log Collector Service
After=network.target

[Service]
ExecStart=/usr/bin/python3 /경로/py파일명  // 내가 적은 경로: /root/Desktop/log_test.py
Restart=always

[Install]
WantedBy=multi-user.target

b. 데몬 재로드 및 해당 서비스 시작

sudo systemctl daemon-reload

sudo systemctl start fortigate_log_collector

sudo systemctl enable fortigate_log_collector

작동 확인 : systemctl status fortigate_log_collector
